\section{Methods}

The methods section should make the analytic pipeline auditable. It should describe the data source, inclusion and exclusion rules, missing-data decisions, feature construction, train-validation-test strategy, candidate models, tuning process, evaluation metrics, and software environment.

Model comparison should include a simple baseline. A complex model is justified when it improves performance, interpretability, calibration, fairness, or scientific utility relative to simpler alternatives.
