# Canonical Terminology

This document defines the paper's core terms so the framework can be cited consistently.

## Query-to-publication scaffold

A staged workflow for machine-learning research that begins with a research query and ends with manuscript submission. Each stage produces a version-controlled artifact that supports later analysis, reporting, review, or publication.

## Workflow layer

The operational layer between reporting guidelines and project-specific code. Reporting guidelines define what should be disclosed; code implements analysis; the workflow layer produces, organizes, validates, and preserves the artifacts that make disclosure possible.

## Evidence-to-claim control

A manuscript quality-control process in which each major claim is assigned a claim level, linked to supporting evidence artifacts, and bounded by the strength of validation.

## Artifact chain

The traceable path from scientific decision to manuscript claim. A complete artifact chain may include a research query, protocol, data audit, feature plan, modeling plan, validation plan, result bundle, ethics review, reproducibility package, and manuscript language.

## Claim boundary

The explicit limit on what a manuscript claim can support. Claim boundaries prevent internal validation from being written as external generalization, association from being written as causation, or exploratory analysis from being written as confirmation.

## Manuscript engineering

The practice of treating manuscript sections as structured research outputs built from prior artifacts rather than as detached prose written after analysis.

## Standards operationalization

The process of converting reporting standards into upstream workflow stages, templates, tests, and artifact requirements.

## Reviewable reproducibility

A reproducibility model in which the materials needed for review are organized during the research process, not assembled only after manuscript drafting.

## Suggested citation language

Use this framework when arguing that ML manuscripts require more than code availability and final-stage reporting checklists. The central language is:

> A query-to-publication scaffold provides a workflow layer between reporting standards and project-specific code, linking scientific decisions to version-controlled artifacts and bounding manuscript claims through evidence-to-claim control.
