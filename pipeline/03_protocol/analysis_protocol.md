# Stage 03 Analysis Protocol

## Study design

Describe whether the project is predictive, explanatory, descriptive, evaluative, or methodological.

## Target estimand or prediction target

State what the model is intended to estimate, predict, classify, rank, or detect.

## Baseline model

Name the simplest defensible baseline model.

## Candidate models

List candidate models before analysis begins.

## Feature plan

Define eligible predictors, excluded predictors, leakage risks, and feature engineering rules.

## Validation design

Specify train, validation, and test design. If cross-validation is used, define folds and grouping rules.

## Metrics

Identify the primary metric and secondary metrics. Explain why each metric matches the research query.

## Sensitivity checks

List planned sensitivity, subgroup, calibration, and robustness checks.

## Reproducibility controls

Record software versions, random seeds, environment files, and expected output locations.

## Exit gate

Proceed when the protocol is specific enough that another researcher could implement the analysis without guessing the major decisions.
