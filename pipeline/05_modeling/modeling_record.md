# Stage 05 Modeling Record

## Baseline results

Record baseline model settings, performance, and interpretation.

## Candidate model runs

For each candidate model, document:

- Model family
- Hyperparameters
- Training data
- Validation data
- Tuning method
- Random seed
- Software version
- Output location

## Model comparison

Compare models against the baseline using the protocol-defined metrics.

## Selection rationale

Explain why the selected model is preferred. Do not rely only on a marginal performance gain if the model is less interpretable, less stable, or harder to reproduce.

## Negative results

Record models or specifications that failed, underperformed, or created instability.

## Exit gate

Proceed when model selection is justified by documented evidence rather than retrospective preference.
