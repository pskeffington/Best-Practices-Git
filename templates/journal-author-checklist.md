# Journal Author Checklist for ML Manuscripts

Use this checklist before submitting a machine-learning manuscript.

## Manuscript package

- [ ] Manuscript source is included.
- [ ] References file is included.
- [ ] Tables and figures are included or reproducibly generated.
- [ ] Funding and conflict-of-interest statements are included.

## Research query and design

- [ ] Research question is stated clearly.
- [ ] Population or sample is defined.
- [ ] Outcome or target is defined.
- [ ] Study design is described.
- [ ] Baseline comparison is justified.

## Data and features

- [ ] Data source is described.
- [ ] Data access restrictions are stated.
- [ ] Inclusion and exclusion criteria are documented.
- [ ] Variable construction is documented.
- [ ] Missing data handling is documented.
- [ ] Feature engineering choices are documented.
- [ ] Leakage risks are reviewed.

## Modeling and validation

- [ ] Candidate models are described.
- [ ] Baseline model is reported.
- [ ] Validation strategy is stated.
- [ ] Metrics match the research question.
- [ ] Sensitivity or diagnostic checks are reported.
- [ ] Subgroup or fairness checks are included when relevant.

## Evidence-to-claim control

- [ ] Each major claim has an evidence source.
- [ ] Each major claim has a claim level.
- [ ] Each major claim has a boundary note.
- [ ] Causal language is avoided unless supported by design.
- [ ] Generalization language is avoided unless supported by validation.
- [ ] Deployment language is avoided unless supported by use-case evidence.

## Reproducibility package

- [ ] Code is included or access instructions are provided.
- [ ] Data dictionary is included.
- [ ] Variable-construction scripts are included.
- [ ] Environment or dependency file is included.
- [ ] Output manifest is included.
- [ ] Audit report is included.
- [ ] Reviewer report is included.
