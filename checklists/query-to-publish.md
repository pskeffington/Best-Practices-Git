# Query-to-Publication Checklist

## 1. Query framing

- [ ] Research question is stated in one paragraph.
- [ ] Population, outcome, predictors, and target use are defined.
- [ ] Machine learning is justified against a simpler baseline.
- [ ] Contribution type is identified: empirical, methodological, translational, or evaluative.

## 2. Data readiness

- [ ] Data source and provenance are documented.
- [ ] Inclusion and exclusion rules are recorded.
- [ ] Missing-data strategy is stated before modeling.
- [ ] Feature construction is reproducible.
- [ ] Sensitive data are excluded from Git history.

## 3. Modeling

- [ ] Baseline model is included.
- [ ] Candidate models are named and justified.
- [ ] Train, validation, and test strategy is documented.
- [ ] Evaluation metrics match the research question.
- [ ] Tuning process is reproducible.

## 4. Validation and interpretation

- [ ] Performance is reported on held-out data or a defensible resampling design.
- [ ] Calibration, uncertainty, or sensitivity checks are included when relevant.
- [ ] Error analysis identifies failure modes.
- [ ] Interpretation does not exceed the validation design.

## 5. Ethics and governance

- [ ] Intended use and non-use are documented.
- [ ] Privacy risks are reviewed.
- [ ] Fairness and subgroup performance are assessed when relevant.
- [ ] Deployment limits are stated.

## 6. Manuscript readiness

- [ ] Abstract states the problem, approach, contribution, and implication.
- [ ] Methods are detailed enough for review.
- [ ] Tables and figures are readable without code.
- [ ] Limitations are specific, not generic.
- [ ] References are verified.

## 7. Publication package

- [ ] Manuscript builds from source.
- [ ] Code runs in a clean environment.
- [ ] Data access instructions are present.
- [ ] Reproducibility materials are archived.
- [ ] Submission journal or venue requirements are checked.
